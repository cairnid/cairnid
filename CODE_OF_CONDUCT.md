# Code of Conduct

Cairn Identity is security-sensitive infrastructure. Project spaces should stay professional, precise, and useful for people trying to build, review, operate, or secure the software.

## Expected Behavior

- Treat other contributors and users with respect.
- Keep technical disagreement focused on evidence, tradeoffs, and project goals.
- Assume security reports and operational concerns are raised in good faith.
- Avoid sharing private information, credentials, exploit details, or sensitive operational data in public channels.
- Make room for maintainers to ask for reproduction steps, threat models, and concrete impact before accepting a change.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing someone else's private information without explicit permission.
- Publicly disclosing unresolved security vulnerabilities in a way that puts users at risk.
- Spam, sustained disruption, or bad-faith issue/PR activity.
- Pressure to weaken security controls, bypass review, or merge unverifiable changes.

## Reporting

For conduct concerns that can be handled without sharing private information, credentials, exploit details, or sensitive operational data, open a repository issue. For vulnerabilities, use GitHub private vulnerability reporting in [SECURITY.md](SECURITY.md) instead of opening a public issue.

Maintainers may remove comments, close issues or pull requests, limit participation, or block accounts when behavior puts the project or community at risk.
